# 身份源管理

身份源管理用于配置企业登录来源，让不同邮箱域名的用户按指定方式登录系统（OIDC 或 LDAP）。

## 一、功能概览

通过身份源管理，你可以统一维护外部身份系统接入规则，降低账号接入与单点登录配置分散带来的风险。

当你需要接入企业 SSO、按域名分流登录、或管控账号冲突处理策略时，身份源管理是核心入口。

### 前提条件

1. 你拥有身份源相关权限（查看 / 新建 / 编辑 / 删除）。
2. 已准备好对应身份系统参数（如 Issuer URL、Client ID、Server URL、Bind DN）。

---

## 二、功能清单

### 功能 1：查看身份源列表

功能介绍：
在列表中查看身份源名称、类型（OIDC / LDAP）、域名、状态与创建时间。

使用场景：
- 例行巡检当前启用的登录来源
- 排查某个邮箱域名用户无法登录的问题

### 功能 2：新建身份源

功能介绍：
支持新建 OIDC 或 LDAP 身份源，并配置域名、账号冲突策略、默认角色 ID、强制 SSO 及对应连接参数。

使用场景：
- 新增组织接入，需按邮箱域名启用企业登录
- 从本地账号登录逐步迁移到统一身份系统

### 功能 3：编辑身份源

功能介绍：
可修改身份源名称、域名、冲突策略、默认角色 ID、强制 SSO 及连接参数。

使用场景：
- 身份系统地址或凭据轮换
- 调整登录后账号处理策略

### 功能 4：启用/停用身份源

功能介绍：
通过状态开关快速启用或停用某个身份源。

使用场景：
- 临时下线异常身份源，避免影响更多用户
- 分阶段灰度启用新身份源

### 功能 5：测试连接

功能介绍：
对已保存的身份源执行“测试连接”，快速验证配置可用性。

使用场景：
- 新建后上线前自检
- 配置调整后回归验证

### 功能 6：删除身份源

功能介绍：
删除不再使用的身份源配置。

使用场景：
- 历史接入下线后清理配置
- 合并重复配置，减少误用

## 三、常见任务操作

### 任务 1：新建 OIDC 身份源并绑定域名

路径：系统管理 > 身份源 > 新建身份源

步骤：
1. 单击“新建身份源”。
2. 选择类型“OIDC”。
3. 填写名称与“域名”（多个域名用逗号分隔）。
4. 选择“账号冲突策略”（拒绝登录 / 自动关联）。
5. 按需填写“默认角色 ID”，并决定是否勾选“强制 SSO 登录”。
6. 在“OIDC 配置”中填写 Issuer URL、Client ID、Client Secret，确认 Callback URL。
7. 按需调整 Scopes，单击“保存”。

结果：
- 身份源创建成功并出现在列表中。

影响：
- 匹配该域名的用户会按新身份源登录。
- 冲突策略会影响账号首次登录时的处理方式。

可撤销性：
- 可编辑、可停用；删除为不可撤销。

### 任务 2：测试已保存身份源连接

路径：系统管理 > 身份源 > 测试连接

步骤：
1. 在目标身份源行单击“测试连接”。
2. 等待系统返回“连接测试成功”或失败提示。
3. 若失败，按错误信息修正配置后重试。

结果：
- 可快速确认当前配置是否可连通。

影响：
- 仅用于验证连接，不直接改动用户数据。

可撤销性：
- 测试操作本身无数据写入，无需撤销。

### 任务 3：停用异常身份源

路径：系统管理 > 身份源

步骤：
1. 在目标身份源“状态”列关闭开关。
2. 确认状态更新。
3. 通知受影响域名用户临时登录策略。

结果：
- 该身份源停止参与登录。

影响：
- 匹配域名的用户可能无法继续通过该来源登录。
- 会影响组织接入与登录体验，应提前评估替代路径。

可撤销性：
- 可再次启用恢复。

### 任务 4：删除废弃身份源

路径：系统管理 > 身份源 > 删除

步骤：
1. 在目标身份源行单击“删除”。
2. 在确认框中确认删除。

结果：
- 身份源被永久移除。

影响：
- 对应域名将不再匹配该身份源。
- 若无替代身份源，相关用户登录可能受阻。

可撤销性：
- 不可撤销；如需恢复需重新创建。

---

## 四、风险与注意事项

- “类型创建后不可更改”，创建前先确认选用 OIDC 还是 LDAP。
- 域名配置应避免与其他身份源冲突，避免登录分流异常。
- 开启“强制 SSO 登录”前，建议先完成小范围验证。
- 调整 Client Secret、Bind Password 后，务必立即执行“测试连接”。
- 删除操作不可撤销，建议先停用观察再删除。
