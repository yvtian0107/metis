# 公告管理

公告管理用于发布和维护系统内公告。你可以集中管理公告内容，确保成员在站内及时看到统一通知。

## 一、功能概览

公告管理把“发布、修改、下线（删除）”放在同一入口，适合处理制度通知、维护公告、变更提醒等需要统一触达的场景。

当你需要快速同步全员信息，或修正已发布公告内容时，可以直接在这里完成。

## 二、功能清单

### 功能 1：查看公告列表

功能介绍：
可查看公告的标题、发布者、发布时间，并按“公告标题”搜索。

使用场景：
- 日常查看当前已发布公告
- 快速定位某条历史公告进行更新或删除

### 功能 2：新建公告

功能介绍：
通过“新建公告”填写“标题”和“内容”后发布新公告。

使用场景：
- 发布临时通知（如系统维护提醒）
- 发布常规运营公告（如制度更新）

### 功能 3：编辑公告

功能介绍：
可对既有公告进行“编辑公告”，更新标题或内容。

使用场景：
- 公告文案有错别字或时间变更
- 需要补充执行细则、联系方式等信息

### 功能 4：删除公告

功能介绍：
可删除不再适用或误发的公告，系统会弹出“确认删除”。

使用场景：
- 误发布公告需要立即下线
- 过期公告清理

## 三、常见任务操作

### 任务 1：发布一条新公告

路径：系统管理 > 公告管理 > 新建公告

步骤：
1. 单击“新建公告”。
2. 填写“标题”（必填）。
3. 填写“内容（可选）”。
4. 单击“发布”。

结果：
- 新公告保存成功，并出现在公告列表。

影响：
- 公告会作为系统内通知内容供成员查看。

可撤销性：
- 可通过“编辑公告”修改；也可删除，删除后不可恢复。

### 任务 2：修订已发布公告

路径：系统管理 > 公告管理 > 编辑公告

步骤：
1. 在目标公告行单击“编辑”。
2. 修改标题或内容。
3. 单击“保存”。

结果：
- 公告内容更新，列表中保留同一条记录。

影响：
- 成员看到的公告内容将更新为最新版本。

可撤销性：
- 可再次编辑回滚文案；不支持一键恢复历史版本。

### 任务 3：删除误发公告

路径：系统管理 > 公告管理 > 删除

步骤：
1. 在目标公告行单击“删除”。
2. 在“确认删除”弹窗中确认。

结果：
- 公告被移除，不再出现在公告管理列表。

影响：
- 对应公告内容不再可见。

可撤销性：
- 不可撤销；如需恢复需重新创建。

## 四、风险与注意事项

- “标题不能为空”，建议标题直接表达主题，便于搜索。
- 删除公告不可撤销，建议先确认是否仍有成员需要参考该公告。
- 公告属于面向全体成员的信息入口，发布前建议进行二次校对。
- 紧急通知建议配合[通知渠道](./channel-management.md)联动检查，避免只发布公告但未完成站外触达。
