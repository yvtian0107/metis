# 角色管理

角色管理用于统一定义权限，并将权限批量分配给用户。通过角色，你可以控制用户“看得到什么、做得了什么、能看哪些数据”。

## 一、功能概览

角色管理用于把权限抽象为可复用的角色模板，支撑“按岗位授权、按角色治理”的权限管理方式。

你可以通过该功能统一配置菜单权限、按钮权限和数据范围，降低逐人授权成本，并提升权限治理一致性。

### 前提条件

1. 你拥有角色管理对应权限（查看/新建/编辑/分配/删除）。
2. 菜单结构已初始化。
3. 如需自定义部门范围，组织架构已配置完成。

---

## 二、功能清单

### 功能 1：新建角色

功能介绍：
用于创建新的权限载体，承接岗位职责和业务边界。

使用场景：
- 新业务线或新岗位上线
- 权限体系从粗粒度向细粒度演进

### 功能 2：编辑角色

功能介绍：
用于维护角色名称、编码和描述，保持角色语义清晰。

使用场景：
- 角色命名规范化
- 职责变化后更新说明

### 功能 3：分配菜单权限与按钮权限

功能介绍：
用于控制“可见入口”和“可执行动作”，是权限配置核心。

使用场景：
- 需要开放某模块访问权限
- 需要收敛高风险操作按钮

### 功能 4：设置数据权限范围

功能介绍：
用于控制用户可查看的数据边界，满足内控与合规需求。

使用场景：
- 按部门隔离数据
- 仅允许查看本人数据
- 指定可见部门做精细化管控

### 功能 5：区分系统角色与自定义角色

功能介绍：
用于识别角色来源与管理边界，避免对系统内置角色做不当治理动作。

使用场景：
- 盘点现有角色时，先区分“系统”与“自定义”
- 制定角色清理计划时，优先处理冗余自定义角色
- 权限治理审计时，快速识别可调整范围

### 功能 6：删除角色

功能介绍：
用于下线废弃角色，减少权限模型冗余。

使用场景：
- 临时角色生命周期结束
- 历史角色合并后清理

## 三、常见任务操作

### 任务 1：新建角色

路径：系统管理 > 角色管理 > 新建角色

步骤：
1. 单击“新建角色”。
2. 填写角色名称、角色编码。
3. 按需填写描述。
4. 单击“保存”。

结果：
- 角色创建成功并进入列表。

影响：
- 本身不直接改变用户权限，需分配给用户后生效。

可撤销性：
- 可删除（系统角色除外）。

### 任务 2：分配菜单与按钮权限

路径：系统管理 > 角色管理 > 权限

步骤：
1. 进入目标角色“权限”。
2. 勾选菜单权限。
3. 勾选对应按钮权限。
4. 单击“保存”。

结果：
- 关联该角色的用户即时按新权限生效。

影响：
- 影响该角色下所有用户可见性与可操作性。

可撤销性：
- 可再次调整。

### 任务 3：设置数据权限范围

路径：系统管理 > 角色管理 > 权限 > 数据权限

步骤：
1. 进入“数据权限”页签。
2. 选择范围：全部数据 / 本部门及子部门 / 仅本部门 / 仅本人 / 自定义部门。
3. 若选“自定义部门”，勾选部门范围。
4. 单击“保存”。

结果：
- 数据查询范围即时按新规则生效。

影响：
- 范围放大可能引入越权风险。
- 范围收紧可能导致业务处理受限。

可撤销性：
- 可重新设置。

### 任务 4：删除角色

路径：系统管理 > 角色管理 > 删除

步骤：
1. 单击“删除”。
2. 在确认框确认。

结果：
- 角色从列表移除。

影响：
- 需同步检查关联用户的权限连续性。

可撤销性：
- 不可撤销。

## 四、风险与注意事项

- 角色应按岗位职责建模，避免按个人建角色。
- 敏感操作权限建议收敛到少数角色。
- 建议定期做角色权限复核，防止权限累积。
