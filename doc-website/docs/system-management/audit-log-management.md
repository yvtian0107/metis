# 审计日志

审计日志用于回溯“谁在什么时间做了什么事”。你可以在同一页面查看“登录活动”和“操作记录”，用于安全巡检、问题排查和合规留痕。

## 一、功能概览

审计日志把关键行为记录集中展示，帮助你快速识别异常登录、误操作和高风险变更。

当出现账号安全事件、权限争议或配置变更追踪需求时，可以先在这里定位时间、操作者与行为摘要，再联动其他模块处理。

## 二、功能清单

### 功能 1：查看登录活动

功能介绍：
用于查看登录相关事件，包括“登录成功”“登录失败”“登出”，并展示时间、用户、IP 地址、设备信息。

使用场景：
- 排查账号是否存在异常登录尝试
- 确认某用户何时退出或重新登录
- 安全巡检时核对来源 IP 与设备分布

### 功能 2：筛选与检索审计记录

功能介绍：
用于通过关键词、事件类型、资源类型、日期范围快速缩小范围，减少手工翻页成本。

使用场景：
- 追查某天发生的集中异常行为
- 定位某类资源（如“用户”“角色”）的变更记录
- 对外部审计提供特定时间窗口的证据

### 功能 3：查看操作记录

功能介绍：
用于查看“创建用户”“设置权限”“更新设置”“踢出会话”等操作行为，识别操作者和行为摘要。

使用场景：
- 核对配置变更责任归属
- 排查业务异常前是否有关键操作
- 复盘系统管理动作对后续状态的影响

## 三、常见任务操作

### 任务 1：排查异常登录

路径：系统管理 > 审计日志 > 登录活动

步骤：
1. 进入“审计日志”页面并保持在“登录活动”页签。
2. 在“事件类型”选择“登录失败”或“登录成功”。
3. 设置开始与结束日期，必要时输入用户名搜索。
4. 查看命中记录中的“IP 地址”“设备”“时间”。

结果：
- 快速定位可疑登录行为与发生时间。

影响：
- 仅查看记录，不会修改账号或权限。

可撤销性：
- 无需撤销（只读操作）。

### 任务 2：追踪一次配置变更

路径：系统管理 > 审计日志 > 操作记录

步骤：
1. 切换到“操作记录”页签。
2. 在“资源类型”中选择目标对象（如“设置”“角色”“用户”）。
3. 输入摘要关键词并执行搜索。
4. 按时间查看“操作者”“操作”“摘要”，确认变更链路。

结果：
- 明确变更由谁执行、何时执行、执行了什么。

影响：
- 仅查看记录，不会回滚已有变更。

可撤销性：
- 审计记录本身不可编辑；如需修正结果，请到对应业务模块执行新操作。

### 任务 3：导出审计结论用于协同处置

路径：系统管理 > 审计日志

步骤：
1. 用日期范围与筛选条件收敛到目标记录。
2. 整理关键字段（时间、用户、操作/事件、IP 地址、摘要）。
3. 将结论同步给相关负责人，联动“用户管理”“会话管理”执行处置。

结果：
- 形成可复核的事件线索，支撑后续处置与复盘。

影响：
- 可能触发跨模块治理动作（停用账号、踢出会话、调整策略）。

可撤销性：
- 结论同步可更新；已执行的处置动作需在对应模块单独回退或补救。

## 四、风险与注意事项

- 审计日志用于追溯，不直接替代处置动作；发现风险后需及时联动相关模块。
- 日期范围过大时检索结果会增多，建议先按时间窗口分段排查。
- “登录失败”应结合 IP、设备、用户名综合判断，避免单一指标误报。
- 保留周期受系统设置影响，超出保留期的历史记录可能已被清理。
