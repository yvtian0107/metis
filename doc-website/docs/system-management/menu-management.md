# 菜单管理

菜单管理用于维护系统导航结构与权限入口。你可以在一个页面里统一维护目录、菜单与按钮，并控制哪些入口显示给哪些角色。

## 一、功能概览

菜单管理帮助你把“页面入口”和“操作按钮”组织成清晰的层级结构，减少权限配置混乱。

当系统新增模块、调整导航、或需要收敛高风险操作入口时，菜单管理是最核心的配置入口之一。

### 前提条件

1. 你拥有菜单管理相关权限（查看 / 新建 / 编辑 / 删除）。
2. 已明确本次变更涉及的模块与角色影响范围。

---

## 二、功能清单

### 功能 1：查看菜单树与层级关系

功能介绍：
以树形结构展示“目录 / 菜单 / 按钮”，便于快速理解系统入口组织方式。

使用场景：
- 上线新模块前，确认放置在哪个目录层级
- 排查“看得到菜单但点不开”或“按钮缺失”问题

### 功能 2：新建菜单项（目录 / 菜单 / 按钮）

功能介绍：
支持按类型创建菜单项，并配置菜单名称、路径、图标、权限标识与上级菜单。

使用场景：
- 新增业务页面，需要在导航中提供入口
- 为页面动作补充按钮级权限控制

### 功能 3：编辑菜单项

功能介绍：
用于调整现有菜单项信息，如名称、路径、权限标识、是否隐藏等。

使用场景：
- 页面路径或命名规范调整
- 权限标识统一治理

### 功能 4：拖拽同级排序

功能介绍：
通过拖拽调整同一层级下菜单顺序，优化用户在侧边栏中的阅读与操作路径。

使用场景：
- 将高频模块前置
- 按业务流程重排入口顺序

### 功能 5：添加子项

功能介绍：
可在目录或菜单下直接“添加子项”，快速补齐层级结构。

使用场景：
- 新增模块需要挂在既有父级下
- 细化已有模块的按钮权限项

### 功能 6：删除菜单项

功能介绍：
用于清理废弃入口和冗余按钮，保持导航与权限模型整洁。

使用场景：
- 历史模块下线后清理入口
- 权限治理中移除不再使用的按钮项

## 三、常见任务操作

### 任务 1：新建一个可见菜单入口

路径：系统管理 > 菜单管理 > 新建菜单

步骤：
1. 单击“新建菜单”。
2. 选择类型“菜单”或“目录”。
3. 填写“菜单名称”。
4. 选择“上级菜单”（可选“顶级菜单”）。
5. 填写“路径”（如 `/users`）、“图标”（可选）、“权限标识”（如 `system:user:list`）。
6. 按需设置“隐藏菜单”。
7. 单击“保存”。

结果：
- 菜单项创建成功，并出现在对应层级位置。

影响：
- 会影响导航可见结构。
- 若角色未分配对应权限，用户仍可能不可见或不可访问。

可撤销性：
- 可编辑、可删除（满足删除条件时）。

### 任务 2：为父级入口添加子项

路径：系统管理 > 菜单管理 > 添加子项

步骤：
1. 在目标父级行单击“添加子项”。
2. 选择子项类型（目录 / 菜单 / 按钮）。
3. 填写子项信息并保存。

结果：
- 子项挂载到目标父级下。

影响：
- 会改变该父级下入口结构，可能影响角色授权颗粒度。

可撤销性：
- 可编辑、可删除。

### 任务 3：调整同级菜单顺序

路径：系统管理 > 菜单管理

步骤：
1. 在同级菜单行左侧拖拽手柄。
2. 将菜单拖到目标位置。
3. 松开后系统自动保存同级排序。

结果：
- 同级菜单显示顺序更新。

影响：
- 改变用户在侧边栏中的浏览顺序，不改变权限本身。

可撤销性：
- 可再次拖拽调整。

### 任务 4：删除菜单项

路径：系统管理 > 菜单管理 > 删除

步骤：
1. 在目标菜单行单击“删除”。
2. 在确认框中确认删除。

结果：
- 菜单项被移除。

影响：
- 若该菜单已用于角色授权，相关入口可能同步消失。
- 当菜单包含子项时，需要先清理子项。

可撤销性：
- 不可直接撤销；如需恢复需重新创建。

## 四、风险与注意事项

- 菜单名称、路径、权限标识建议统一命名规范，避免后续治理成本上升。
- 删除前先确认是否存在子项与角色依赖，避免误删导致业务入口中断。
- “隐藏菜单”仅影响入口展示，不等同于移除权限策略。
- 导航调整后，建议同步抽查关键角色的实际可见与可操作结果。
