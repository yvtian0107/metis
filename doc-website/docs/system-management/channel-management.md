# 通知渠道

通知渠道用于配置系统消息的发送出口。你可以在这里维护邮件通道，并在启用前完成连接测试与测试发送，降低通知失败风险。

## 一、功能概览

通过通知渠道，你可以把“系统事件”与“实际发送能力”连接起来，统一管理发送配置、状态与验证动作。

当你需要上线通知、排查收不到消息、或做邮箱参数轮换时，通知渠道是核心入口。

### 前提条件

1. 你拥有通知渠道相关权限（查看 / 新建 / 编辑 / 删除）。
2. 已准备好 SMTP 参数（SMTP 服务器、端口、用户名、密码、发件人）。

---

## 二、功能清单

### 功能 1：查看渠道列表

功能介绍：
在列表中查看通道名称、类型、状态、创建时间，并按名称搜索。

使用场景：
- 日常巡检当前有哪些可用通道
- 快速定位某个发送失败通道

### 功能 2：新建渠道

功能介绍：
支持新建“邮件 (SMTP)”通道，填写连接配置后保存。

使用场景：
- 首次上线系统通知
- 新增业务线需要独立发件通道

### 功能 3：编辑渠道

功能介绍：
可修改通道名称与连接配置。已创建通道的“通道类型”不可更改。

使用场景：
- SMTP 地址、端口或账号密码变更
- 发件人名称调整

### 功能 4：启用/停用渠道

功能介绍：
通过状态开关快速启用或停用某个通道。

使用场景：
- 临时下线异常通道，防止持续发送失败
- 新通道灰度启用

### 功能 5：测试连接

功能介绍：
在编辑状态下执行“测试连接”，验证当前通道配置是否可连通。

使用场景：
- 新配置保存后上线前自检
- 参数调整后的回归验证

### 功能 6：测试发送

功能介绍：
通过“测试发送”填写收件人、主题、内容，发送一封测试邮件。

使用场景：
- 验证连通后进一步确认真实发送链路
- 排查“可连接但收件失败”的问题

### 功能 7：删除渠道

功能介绍：
删除不再使用的通道配置。

使用场景：
- 历史废弃通道清理
- 避免误用过期配置

## 三、常见任务操作

### 任务 1：新建邮件通知渠道

路径：系统管理 > 通知渠道 > 新建通道

步骤：
1. 单击“新建通道”。
2. 填写“通道名称”。
3. 选择“通道类型”为“邮件 (SMTP)”。
4. 在“连接配置”中填写 SMTP 服务器、端口、SSL/TLS、用户名、密码、发件人。
5. 单击“保存”。

结果：
- 新通道创建成功并出现在列表中。

影响：
- 该通道可被后续通知能力引用用于发送消息。

可撤销性：
- 可编辑、可停用；删除不可撤销。

### 任务 2：发送测试邮件验证通道

路径：系统管理 > 通知渠道 > 测试发送

步骤：
1. 在目标通道操作区单击“测试发送”。
2. 输入“收件人”“主题”“内容”。
3. 单击“发送”。
4. 观察提示“测试邮件发送成功”或失败原因。

结果：
- 可确认收件链路是否正常。

影响：
- 会向填写的收件人发送真实测试邮件。

可撤销性：
- 邮件发送后不可撤回。

### 任务 3：停用异常渠道

路径：系统管理 > 通知渠道

步骤：
1. 在目标通道“状态”列关闭开关。
2. 确认状态已更新。
3. 同步相关管理员，避免继续依赖该通道发送。

结果：
- 该通道被停用。

影响：
- 依赖该通道的通知可能无法继续送达。

可撤销性：
- 可再次启用恢复。

### 任务 4：删除废弃渠道

路径：系统管理 > 通知渠道 > 删除

步骤：
1. 在目标通道操作区单击“删除”。
2. 在确认框中确认删除。

结果：
- 通道被永久移除。

影响：
- 该配置不可再用于通知发送。

可撤销性：
- 不可撤销；如需恢复需重新创建。

---

## 四、风险与注意事项

- “通道类型创建后不可更改”，创建前先确认类型。
- 修改密码后建议立即执行“测试连接”与“测试发送”。
- “测试发送”会产生真实邮件，请使用可控收件人地址。
- 停用前先确认是否有流程仍依赖该通道，避免关键通知中断。
- 删除操作不可撤销，建议先停用观察再删除。
