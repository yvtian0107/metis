# 任务管理

任务管理用于查看系统后台任务运行情况，并在需要时进行“暂停”“恢复”“触发”等操作。你可以用它快速判断任务是否健康、是否按预期执行。

## 一、功能概览

任务管理把“定时任务”和“异步队列任务”放在同一入口，方便你统一巡检运行状态、执行结果和失败情况。

当你需要处理任务堆积、排查执行失败、或临时手动触发任务时，这个模块是最直接的操作入口。

## 二、功能清单

### 功能 1：查看任务总览

功能介绍：
可查看“总任务数”“运行中”“今日完成”“今日失败”等指标，并在列表中看到任务名称、类型、状态与上次执行情况。

使用场景：
- 日常巡检任务整体健康度
- 快速发现失败任务或异常增长

### 功能 2：按任务类型筛选

功能介绍：
支持在“全部 / 定时任务 / 异步队列”之间切换，快速聚焦某一类任务。

使用场景：
- 只排查定时任务未按时执行问题
- 只关注异步任务消费状态

### 功能 3：暂停与恢复定时任务

功能介绍：
对“定时任务”可执行“暂停”与“恢复”，用于临时停止或恢复自动执行。

使用场景：
- 变更窗口临时停跑任务
- 故障恢复后重新启用调度

### 功能 4：手动触发任务

功能介绍：
可通过“触发”立即执行任务，并在确认框中执行“执行”动作。

使用场景：
- 需要立即补跑一次任务
- 验证配置调整后的执行结果

### 功能 5：查看任务详情与执行历史

功能介绍：
进入任务详情后可查看任务配置（如 Cron 表达式、超时、最大重试）与执行历史（触发方式、状态、耗时、错误信息、时间）。

使用场景：
- 排查任务持续失败原因
- 复核“手动触发”是否成功落地

## 三、常见任务操作

### 任务 1：巡检今日失败任务

路径：系统管理 > 任务管理

步骤：
1. 进入“任务管理”页面。
2. 查看“今日失败”指标。
3. 在列表中定位状态为“失败”“超时”“中断”的任务。
4. 单击任务名称进入详情，查看最近执行记录与错误信息。

结果：
- 可确认失败任务范围与主要错误线索。

影响：
- 仅查看信息，不修改任务执行状态。

可撤销性：
- 无需撤销（只读操作）。

### 任务 2：暂停一个定时任务

路径：系统管理 > 任务管理 > 目标任务

步骤：
1. 在“定时任务”或“全部”列表中定位目标任务。
2. 单击“暂停”。
3. 确认任务状态变为“已暂停”。

结果：
- 该定时任务停止自动调度执行。

影响：
- 后续计划执行会被中止，可能导致数据同步或清理延后。

可撤销性：
- 可通过“恢复”重新启用。

### 任务 3：手动触发任务并确认结果

路径：系统管理 > 任务管理 > 触发

步骤：
1. 在目标任务操作区单击“触发”。
2. 在“手动触发任务”确认框中单击“执行”。
3. 进入任务详情，查看执行历史中的最新记录。

结果：
- 任务立即执行，并生成一条新的执行记录。

影响：
- 可能触发真实业务处理（如清理、同步、回调处理）。

可撤销性：
- 触发动作本身不可撤销；如产生业务影响，需在对应业务模块补救。

## 四、风险与注意事项

- “暂停”会阻断后续自动执行，操作前先确认是否影响关键业务时效。
- “触发”会立即执行真实任务，建议先确认执行窗口与依赖条件。
- 任务详情中的“错误信息”是排障入口，但修复动作通常需要联动其他模块处理。
- 执行历史保留周期受系统设置影响，超出保留天数的记录可能已被清理。
