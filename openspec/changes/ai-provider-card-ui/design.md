## Context

当前 AI 供应商管理页（`web/src/apps/ai/pages/providers/index.tsx`, 548 行）使用标准数据表格 + 行内可展开模型列表。表格承载了供应商 CRUD、连接测试、模型同步、模型 CRUD 全部功能，导致单个页面职责过重。

供应商数量通常 1-5 个（极少超过 10），表格布局信息密度低且视觉辨识差。用户核心诉求是「一眼看出所有供应商健康状况」+「在某个供应商下管理模型」。

现有组件库完备：Card（含响应式 grid header）、Avatar（size variants + badge）、Collapsible、DropdownMenu、Badge、Tooltip、Separator 等均可用。设计 token 使用 oklch 色彩空间，圆角 0.75rem。

## Goals / Non-Goals

**Goals:**
- 将供应商列表从表格重构为品牌化卡片网格，一眼区分不同供应商
- 拆分信息层级：列表页负责总览和导航，详情页负责完整管理
- 利用品牌色系、状态指示灯、微交互提升视觉品质
- 保持现有后端 API 不变，纯前端重构

**Non-Goals:**
- 不涉及后端 API 或数据模型修改
- 不添加供应商健康自动巡检/定时轮询（可作为后续 change）
- 不重新设计模型的 Sheet 表单（ModelSheet 保持不变）
- 不添加供应商 Logo/SVG 资产（用首字母 Avatar + 品牌色替代）

## Decisions

### 1. 卡片网格替代表格

**选择**: 响应式 CSS Grid（`grid-template-columns: repeat(auto-fill, minmax(340px, 1fr))`）

**理由**: 供应商数量极少（1-5 个），表格的行对比优势无用武之地。卡片可承载更丰富的视觉信息（品牌色、状态灯、统计 chips），且在不同屏幕宽度下自适应。

**替代方案**: 保持表格但增加 Logo 列 → 信息密度仍然低，且无法展示品牌个性。

### 2. 品牌色方案（无 Logo 资产）

**选择**: 顶部 3-4px 色条 + 首字母 Avatar（品牌背景色）

| Provider   | 色条/Avatar bg    | Avatar text       |
|------------|-------------------|-------------------|
| openai     | emerald-500       | emerald-700 on emerald-50 |
| anthropic  | amber-500         | amber-700 on amber-50     |
| ollama     | sky-500           | sky-700 on sky-50         |
| (fallback) | primary           | primary on primary/10     |

**理由**: 无需引入外部 SVG/图片资产，颜色是前注意加工（pre-attentive），识别速度比阅读文字快。3-4px 色条点到为止不会让配置页显得过于花哨。

**替代方案**: 引入供应商 SVG Logo → 维护成本高，需跟随供应商品牌更新。

### 3. 两层信息架构（列表 + 详情页）

**选择**: `/ai/providers` 卡片列表 + `/ai/providers/:id` 详情页

**理由**:
- 卡片内展开模型列表会让卡片高度剧烈变化，破坏网格布局
- 模型管理（表格、搜索、筛选、CRUD）需要足够的水平空间
- 编辑供应商往往伴随测试连接 → 同步模型的完整工作流，详情页能承载全流程

**替代方案**: 卡片内 Collapsible 展开 → 适合 3-5 个模型，但 15+ 个模型时卡片过高；全功能 Drawer → 宽度受限，模型表格挤不下。

### 4. 创建保留 Drawer，编辑移到详情页

**选择**: ProviderSheet 仅用于创建（4 字段简单表单），编辑功能在详情页内完成

**理由**: 创建是一次性操作（填 4 个字段关掉），Drawer 交互路径最短。编辑往往需要确认当前状态、测试连接、同步模型，这些上下文在详情页天然可见。

### 5. 全量加载不分页

**选择**: `pageSize=100`，不渲染分页组件

**理由**: 供应商数量天然有限（没人会配 50 个供应商），全量加载无性能问题。卡片网格下分页体验差——翻页后空间位置重排，用户失去空间记忆。

### 6. 状态指示灯设计

**选择**: 8px 圆点 + CSS 动画

| Status   | 颜色          | 动画                |
|----------|---------------|---------------------|
| active   | green-500     | subtle pulse (2s)   |
| inactive | gray-400      | none                |
| error    | red-500       | faster pulse (1.5s) |

测试中状态: 灰点 → 旋转 Spinner → 结果色点，配合 toast 反馈。

### 7. 操作按钮分层

**选择**: 高频操作（测试连接）常驻卡片底部可见，低频操作（编辑、删除）收进 ⋯ DropdownMenu

**理由**: 渐进式披露原则。测试连接是创建后和日常巡检的第一操作，需要零点击可达。

## Risks / Trade-offs

- **[卡片 vs 表格信息对比]** 卡片布局下同时对比多个供应商的某个字段（如 baseUrl）不如表格直观 → 供应商数量极少（1-5），对比需求弱，可接受
- **[新增路由]** 从单页变为两页，增加了导航复杂度 → 面包屑 + 返回按钮保证导航不迷失
- **[品牌色硬编码]** 供应商类型增加时需手动添加色彩映射 → 提供 fallback 色（primary），新增类型时更新映射 map 即可
- **[全量加载上限]** 极端情况下供应商超过 20 个 → 保留 useListPage 的分页能力但默认 pageSize=100，未来需要时可切回分页
